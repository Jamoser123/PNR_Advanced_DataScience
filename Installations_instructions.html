Windows

Follow the steps below to install Python, set up a virtual environment, and install the required packages:

Step 1: Download and Install Python

  1. Check if Python is Already Installed:
    • Open Command Prompt (press Windows + R, type cmd, and press Enter).
    • First, check the Python version by typing: 
      python --version
    • If Python is not found, try checking with the py command: 
      py --version
    • If either command returns a version number, Python is already installed, and you can skip to Step 2.
    • If both commands return an error like "Python not found", proceed to the next step to install Python.
  2. If Python is Not Installed, Download Python:
    • Visit the official Python website: python.org.
    • Download the latest version of Python for Windows.
  3. Run the Installer:
    • Open the Python installer.
    • Important: Check the box that says "Add Python to PATH" at the bottom of the installer window. This ensures Python and pip can be accessed from the Command Prompt.
    • Click "Install Now" and wait for the installation to complete.
  4. Verify Python Installation:
    • Open Command Prompt again and check the Python version by typing: 
      python --version
  5. If This Doesn’t Work:
    • If you receive an error like "Python not found", you'll need to add Python to the system's PATH manually (explained below). Alternatively, you can use the py command (explained in Step 5).

Step 2: Add Python to PATH (If Not Done During Installation)

If Python isn't recognized by the python command (or the py command), you may need to manually add Python to your system’s PATH.

  1. Find Python Installation Directory:
    • Navigate to the directory where Python was installed. Common locations are:
      • C:\Users\\AppData\Local\Programs\Python\Python\
      • Or C:\Program Files\PythonXX\ (replace XX with your version number).
  2. Add Python to PATH:
    • Right-click the Start button and select System.
    • In the System window, click Advanced system settings on the right.
    • In the System Properties window, click Environment Variables.
    • Under System variables, scroll down and find Path, then click Edit.
    • Click New, and paste the full path to the folder where python.exe is located (e.g., C:\Users\\AppData\Local\Programs\Python\Python).
    • Click New again, and add the path to Python's Scripts folder (e.g., C:\Users\\AppData\Local\Programs\Python\Python\Scripts).
  3. Save and Restart:
    • Click OK to close all windows and save the changes.
    • Restart Command Prompt and try running: 
      python --version

Step 3: Use py Instead of python (If python Does Not Work)

If, after setting the PATH, python --version still doesn’t work, Windows often installs Python with the py launcher. You can use py instead of python for all Python-related commands.

  • To check the Python version, use: 
    py --version
  • To start Python using py, simply type: 
    py

Step 4: Verify pip Installation

pip is the package manager for Python, and it’s installed automatically with Python. However, you can verify that pip is working properly.

  1. Check pip:
    • Open Command Prompt, and type: 
      pip --version
    • If you see a version number, pip is installed and working.
  2. If pip is Not Installed:
    • Download get-pip.py from https://bootstrap.pypa.io/get-pip.py.
    • Open Command Prompt, navigate to where you downloaded get-pip.py, and run: 
      python get-pip.py
      If python doesn’t work, use the following command instead: 
      py get-pip.py
  3. Verify pip Installation Again:
    • After installing pip, you can verify it by running: 
      pip --version

Step 5: Create a Virtual Environment

Once Python and pip are installed, it's good practice to create a virtual environment for your project to avoid conflicts with other Python projects.

  1. Create the Virtual Environment:
    • Open Command Prompt, navigate to your project folder, and run: 
      python -m venv datascience
      If python doesn’t work, use the following command instead: 
      py -m venv datascience
  2. Activate the Virtual Environment:
    • On Windows, activate the environment by running: 
      datascience\Scripts\activate
    • You should now see (datascience) appear before your command prompt, indicating the virtual environment is active.

Step 6: Install Packages from requirements.txt

Once your virtual environment is set up, you can install all the necessary packages from a requirements.txt file. This file contains a list of all the required packages for your project.

  1. Ensure the Virtual Environment is Activated: (you should see (datascience) before the prompt).
  2. Install Packages:
    • Before installing, make sure you're in the directory containing your requirements.txt file. You can use the cd (change directory) command in the terminal to navigate to that folder. For example, if your file is located in the "Documents" folder:
      • cd ~/Documents
    • Once you are in the correct folder, run the following command to install all the packages in the datascience environment:
      • pip install -r requirements.txt

Step 7: Deactivate the Virtual Environment (Optional)

Once you're done working in the virtual environment, you can deactivate it to return to your system's default Python environment.

  1. Deactivate:
    • To deactivate the virtual environment, simply type: 
      deactivate

Additional Notes

  • The virtual environment isolates your project's dependencies, so they don’t conflict with other Python projects you may have on your system.
  • After activating the virtual environment and installing the packages, you can work on your project without worrying about interfering with other projects or system-wide Python packages.

macOS

Step 1: Install Python

macOS comes with Python 2 pre-installed, but you’ll need Python 3.

  1. Check if Python 3 is installed:

    Open Terminal and check if Python is installed by typing:

    python3 --version

    If Python 3 is already installed, you’ll see the version number.

    If Python 3 is not installed, proceed to the next step to install it.

  2. If Python is Not Installed, Install Python 3: The recommended way to install Python 3 is via Homebrew. If you don’t have Homebrew installed, first install it by running: 
    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
    Then, install Python by running: 
    brew install python
  3. Verify the Installation: Confirm Python 3 and pip3 are installed: 
    python3 --version
    pip3 --version

Step 2: Create New Environment and Install Packages

  1. Install virtualenv if not installed: Install virtualenv with pip: 
    pip3 install virtualenv
  2. Create the datascience environment: Create a new virtual environment named datascience: 
    python3 -m venv datascience
  3. Activate the environment: Activate the virtual environment: 
    source datascience/bin/activate  # Linux/macOS
  4. Install all required packages from requirements.txt:

    First, navigate to the folder where your requirements.txt file is located. You can do this using the cd (change directory) command in the terminal. For example, if your file is located in the "Documents" folder:

    cd ~/Documents

    Once you are in the correct folder, run the following command to install all the packages in the datascience environment:

    pip install -r requirements.txt
  5. Verify the installations: To confirm everything was installed correctly, use: 
    pip list

Optional: Verify Everything is Working

  1. Run a quick Python command to ensure everything is working: 
    python -c "print('Python is working!')"
  2. Start Jupyter Notebook: 
    jupyter notebook

Linux

Follow the steps below to install Python, set up a virtual environment, and install the required packages:

Step 1: Install Python

  1. Install Python via Package Manager:
    • Open your terminal and use the appropriate package manager for your distribution to install Python.
    • For Ubuntu/Debian-based systems, run: 
      sudo apt update && sudo apt install python3 python3-pip python3-venv
    • For Fedora-based systems, use: 
      sudo dnf install python3 python3-pip python3-venv
    • For Arch-based systems, use: 
      sudo pacman -S python python-pip python-virtualenv
  2. Verify Python Installation:
    • Once installed, check the Python version by typing: 
      python3 --version

Step 2: Verify pip Installation

pip is the package manager for Python, and it’s installed automatically with Python. You can verify if pip is working with the following steps:

  1. Check pip:
    • Open your terminal, and type: 
      pip3 --version
    • If you see a version number, pip is installed and working.
  2. If pip is Not Installed:
    • Run the following command to install pip: 
      sudo apt install python3-pip

Step 3: Create a Virtual Environment

Once Python and pip are installed, it's a good practice to create a virtual environment for your project to avoid conflicts with other Python projects.

  1. Create the Virtual Environment:
    • Navigate to your project folder in the terminal and run: 
      python3 -m venv datascience
  2. Activate the Virtual Environment:
    • To activate the virtual environment, run: 
      source datascience/bin/activate
    • You should now see (datascience) appear before your command prompt, indicating the virtual environment is active.

Step 4: Install Packages from requirements.txt

Once your virtual environment is set up, you can install all the necessary packages from a requirements.txt file. This file contains a list of all the required packages for your project.

  1. Ensure the Virtual Environment is Activated: (you should see (datascience) before the prompt).
  2. Install Packages:
    • In the same directory as requirements.txt, run: 
      pip install -r requirements.txt

Step 5: Deactivate the Virtual Environment (Optional)

Once you're done working in the virtual environment, you can deactivate it to return to your system's default Python environment.

  1. Deactivate:
    • To deactivate the virtual environment, simply type: 
      deactivate

Additional Notes

  • The virtual environment isolates your project's dependencies, so they don’t conflict with other Python projects you may have on your system.
  • After activating the virtual environment and installing the packages, you can work on your project without worrying about interfering with other projects or system-wide Python packages.
